[Fix] Proses Boot Stuck di Reached Target System Update

Kemarin Shubuh muncul pemberitahuan auto-update dari GNOME Software, yang ujung-ujungnya minta restart buat install, setelah saya restart dan ditinggal ke Puskesmas + Radio Visit demi kepentingan GNOME Release Party 3.18, ternyata masih belum selesai juga proses boot dan stuck di tampilan

[OK] Reached Target System Update

Lumayan nambah-nambah kerjaan nih.

Googling, ketemu masalah yang sama di forum fedora[0] dengan cara menghapus folder /var/cache, lalu saya reboot, pindah ke Fedora 22 yang kebetulan terinstall dengan sengaja sebagai 'backup' OS. Setelah menghapus folder /var/cache dari partisi arch, ternyata malah nambah masalah lain, yaitu stuck di

[OK] Reached Target Timers

Apa-apaan ini?

Setelah dipikir beberapa menit, saya kepikiran kalau masalah ini muncul karena auto-update, seperti di forum fedora[0] tersebut, saya berpikir bahwa system stuck karena auto-update tersendat dan saya harus update manual. Masuk akal.

Kembali boot ke Fedora, lalu saya dump disk file installer (ISO) archlinux ke USB Flash Disk, cara untuk menduplikat file ISO ke UFD cukup mudah, begini

$ sudo dd if=/folder/file.iso of=/dev/sdX

Setelah UFD siap, reboot system lalu boot ke UFD via BIOS (laptop saya masih menggukanan BIOS) lalu pilih arch install yang sesuai dengan arsitektur sebelumnya. Jadi, misal dulu install arch 32 bit, pilihlah yang i386, tapi jika dulu install arch 64 bit, pilih yang x86_64.

Setelah masuk ke konsol arch installer, mount partisi harddisk dimana arch terinstall, punya saya di /dev/sda2

# mount /dev/sda2 /mnt

Setelah partisi bisa dibuka, akses partisi tersebut dengan hak akses root, selanjutnya update system di partisi tersebut.

# arch-chroot /mnt
# pacman -Syu

Jika sampai langkah di atas selesai, hal terakhir yang perlu dilakukan adalah reboot ke HDD dan jalankan arch seperti biasa. Dengan cara ini, saya langsung bisa masuk ke system dengan normal.

Link:
[0] https://ask.fedoraproject.org/en/question/54667/boot-hangs-at-reached-target-system-update/

Fix Webcam ASUS X200CA

Masalah datang ke meja, ketika adiknya Siska bawa ASUS X200CA dengan webcam yang gak bisa nyala, gak konek gitu, Cheese aja ngeblank (wkwkwkwkwk)

Kalau bingung seperti apa ASUS X200CA, bisa buka ke halaman resmi ASUS[0]

Karena saya yang install manjaro di netbooknya dia, saya jadi merasa bertanggung jawab. Baiklah, googling semalaman ternyata internet Telkomsel lemot banget. Akhirnya hari ini bisa ngerjain ini webcam!

Begini cara saya memecahkan masalah

1. Cek BIOS

Siapa tau webcam dinonaktifkan dari BIOS, maka cek dulu konfigurasi hardware di BIOS, masuk BIOS bisa dengan tombol F2 atau Delete. Sudah masuk BIOS? Umumnya, konfigurasi untuk webcam ada di menu Advance dan bisa berada di sub-menu yang berbeda, di ASUS X200CA ini saya lupa di mana :D

Ternyata di BIOS sudah Unlock, artinya sudah diaktifkan, tapi masih belum bisa tersambung ke Cheese, langkah selanjutnya . . .

2. Cek Hardware

Biasanya dengan perintah lsusb di terminal akan menampilkan informasi webcam, kalau dari beberapa forum dan website sih begitu, tapi di sini nihil.

Cara terakhir menggunakan lshw. Karena di manjaro tidak ada lshw, kita perlu install lshw terlebih dahulu.

$ sudo pacman -S lshw
$ sudo lshw > hardware.txt

Kalau sudah selesai, buka file hardware.txt dengan file editor favorit, lalu cari informasi webcam, di kasus ini saya menemukan USB2.0 UVC Camera, sepertinya webcam generik.

3. Penyelesaian

Meluncur ke archwiki[1] lalu cari keyword uvc, di sana ada paket linux-uvc yang ternyata sudah terinstall di Manjaro, lalu cari lagi dan menemukan guvcview dan wajib dicoba.

$ sudo pacman -S guvcview

Setelah guvcview terinstall, coba buka aplikasi guvcview dari application menu, jangan panik jika masih blank, coba akses menu Setting -> Camera Select (kalau tidak salah) dan secara ajaib aplikasi guvcview langsung bisa mengakses kamera.

Ini hasil akhirnya

Coba buka aplikasi Cheese

BISA!

Link:

[1] https://wiki.archlinux.org/index.php/Webcam_setup

Swap Tombol Alt dan Ctrl ala Mac Keyboard.

Niatnya cuma iseng, pengen coba ngetik ala pengguna macintosh, macbook, atau simplenya mac keyboard. Emang gimana sih? Jadi kalau di keyboard mac, tombol Alt dan Ctrl itu tukeran tempat, alias tombol Alt ada di tombol Ctrl dan sebaliknya. Jadi biar saya tampilkan gambarnya.

Ini keyboard English (US) standar

[caption id="attachment_4001" align="alignnone" width="724"] Keyboard Standar en_US[/caption]

Ini keboard Mac English (US)

[caption id="attachment_4002" align="alignnone" width="700"] Keyboard Mac en_US[/caption]

Sebagai catatan, cara ini baru berlaku untuk keyboard standar, untuk laptop beda lagi karena ada tombol Fn yang letaknya beda, laptop saya tombol Fn diapit tombol Ctrl dan tombol Super, untuk keyboard Macbook sendiri tombol Fn berada paling luar.

Yang diperlukan adalah membuat file .Xmodmap dengan perintah $ gedit ~/.Xmodmap dan isikan konfigurasi ini:

clear control
clear mod1
clear mod4

! LEFT
keycode 37 = Alt_L Meta_L
keycode 64 = Control_L


! RIGHT
keycode 108 = Control_L
keycode 105 = Alt_L Meta_L

! ADD CONTROL
add control = Control_L Control_R
add mod1 = Alt_L Alt_R Meta_L Meta_R

Jika sudah selesai, reload konfigurasi xmodmap dengan perintah $ xmodmap ~/.Xmodmap

Selamat mencoba, dan selamat menyesuaikan diri :)

It's Time to Stay

Saya sering pindah-pindah desktop sampai harus pindah distro, dari mulai Ubuntu (hampir semua varian sudah dicoba), Debian, Crunchbang, openSuSE, Fedora, Manjaro, Puppy Linux, Arch, dan masih banyak yang lupa apa saja, mungkin masih ada 5 yang belum kesebut, entahlah.

Setelah baca tulisan ini https://restava.wordpress.com/2015/06/19/anda-berani-memilih/, akhirnya sadar, bahwa pindah-pindah distro itu melelahkan, harus install lagi, harus install lagi, install lagi, belum lagi konfigurasi ini itu, belum lagi ngatur ini itu, emang gak capek?

Terus, sudah mengambil keputusan? Oh sudah, saya memutuskan untuk menetap di Arch Linux. Tepuk tangan dulu dong..!

[caption id="attachment_3994" align="alignnone" width="338"] Tepuk tangan![/caption]

Kenapa, Ada Apa dengan Arch?

  1. Cara install-nya gak biasa, entah kenapa suka sama cara install arch yang sudah jarang ditemukan di distro modern, yep, text mode, install via terminal.
  2. Rolling Stones Release, jadi kalau nanti ada major update, gak perlu install baru atau nimpa versi sebelumnya, cukup update seperti biasa pake sudo pacman -Syu.
  3. Wiki lengkap, solving di forum juga sudah banyak, bahkan saya sering menemukan solusi dari forum arch untuk masalah yang ditemukan di distro lain.
  4. AUR atau ArchLinux User Repository, sebuah repository yang dikelolah oleh komunitas, isinya cukup lengkap, bahkan kemarin saya install atom dari AUR ini. Yang saya suka dari AUR dibanding PPA adalah, saya tidak perlu menambahkan repository list baru untuk masing-masing developer, misal saya pengen install atom editor, kalau di Ubuntu, saya harus begini dulu 
    sudo add-apt-repository ppa:webupd8team/atom
sudo apt-get update
sudo apt-get install atom

    kalau via aur, cukup begini

    yaourt atom-editor

    lalu pilih paket yang mana yang akan di-install, karena dalam beberapa kasus, ada lebih dari satu paket yang dikelola oleh tim yang berbeda, atom-editor aja ada dua pilihan.
    Perlu diingat, install dari AUR dengan perintah yaourt tidak boleh menggunakan sudo di awal, kalau nanti di pertengahan proses minta password, isikan di situ.

  5. Stabil, ringan, dan up-to-date. Yoi, ini stabil, jarang saya nemu pesan error ketika login ke desktop, atau tiba-tiba freeze gak jelas karena kehabisan resource, jika dibandingkan dengan distro Ubuntu, Arch lebih stabil saya rasa. Dan nilai lebihnya, perangkat lunak yang tersedia di pacman dan aur up-to-date, jadi tidak perlu menunggu terlalu lama atau mengambil resiko menggunakan repo testing / beta untuk penggunaan harian jika ingin mencoba perangkat lunak versi baru.

Hindari Install Arch Linux Jika

  1. Tidak punya koneksi internet dan tidak punya panduan instalasi, misal dicetak atau browsing di HP. Oh iya, punya device cadangan yang terkoneksi internet itu wajib, karena kalau ada masalah, kita bisa browsing di device cadangan untuk cari solusi.
  2. Baca lagi point di atas.

Install Raspbian di Raspberry Pi, Mainan Baru :D

Malam ini cukup membahagiakan, setelah berkunjung ke kosan Fahmi dan dia curhat galau Tugas Akhir yang tak kunjung selesai, akhirnya saya berniat membantunya, dan yang membuat saya bahagia adalah saya mendapat Raspberry Pi A yang sudah lama saya idamkan tapi belum ada rezeki untuk memboyong satu unit ke kosan.

Tugasnya 'cuma', (gak mau meremehkan sih) bikin script PHP yang bisa print langsung ke printer via raspi, kenapa? Karena web servernya nanti ada di raspi, begitu ceritanya.

Nah, saya dapet raspi A kosongan, belum install apa-apa bahkan SD Card yang Fahmi kasih belum ada OSnya. Baiklah.

Sekarang yang akan saya ulas adalah cara install Sistem Operasi di Raspberry Pi, saya menggunakan Raspbian Wheezy.

  1. Download file Raspbian dari sini https://www.raspberrypi.org/downloads/
  2. Extract file yang telah didownload tadi, cari file dengan ekstensi .img lalu rename menjadi raspbian.img untuk memudahkan proses selanjutnya.
  3. Masukkan SD Card ke USB Card Reader / Card Reader yang tersedia di laptop (notebook / netbook) lalu cek lokasi SD Card, misal di /dev/sdb
  4. Buka aplikasi terminal di linux lalu buka folder di mana file raspbian.img tadi disimpan dengan perintah cd.
  5. Ketikan perintah sudo dd if=raspbian.img of=/dev/sdb untuk mengekstrak raspbian ke SD Card.
  6. Setelah proses dump disk selesai, pindahkan SD Card ke slot yang tersedia di Raspberry Pi.
  7. Nyalakan Raspberry Pi.
  8. Langkah selanjutnya adalah konfigurasi Raspbian, tidak perlu melakukan partisi karena sudah mengikuti pengaturan dari Raspbian, yang saya lakukan hanya mengganti password. Dialog ini bisa dimunculkan kembali dengan perintah sudo raspi-config
  9. Klik tombol Finish di dialog untuk menyelesaikan konfigurasi, lalu system akan merestart sendiri.

Redeem DigitalOcean Coupon dari GitHub Education Pack

Sudah dapat GitHub Education Pack? Kalau sudah, kita bisa ambil offering dari GitHub dan sponsor lain. Caranya? Buka https://education.github.com/pack/ (Kalau belum, bisa buka post saya di sini GitHub Education Pack untuk meminta diskon atau hak untuk offering dari GitHub).

DigitalOcean sendiri adalah penyedia Virtual Private Server (VPS) yang juga menjadi sponsor GitHub Education.

Lanjut saja, begini caranya:

  1. Buka GitHub Education Pack di https://education.github.com/pack/ lalu klik "Get your pack".

    [caption id="attachment_3957" align="alignnone" width="700"] Request Kode Kupon[/caption]
